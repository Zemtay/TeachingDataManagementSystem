# 智能学生课程管理系统开发文档

## 一、项目简介
本项目基于 RuoYi 框架开发，采用 **Spring Boot (Java) + Vue** 前后端分离架构。核心目标是实现学生、课程、班级的全流程智能管理，集成用户认证、权限控制、教务数据统计分析、AI 辅助功能等模块，支持学校/培训机构高效开展教学管理工作，提升课程安排、学生跟踪、成绩分析的智能化水平。

前端主界面突出“教学数据可视化决策辅助”，强调通过智能化分析、自然语言交互和多角色管理提升教学数据处理效率。（见[ruoyi-ui/src/views/index.vue](https://github.com/Zemtay/TeachingDataManagementSystem/blob/53f6ba0eefbea296a0ca5e1310c64e3bffe92c22/ruoyi-ui/src/views/index.vue)）

---

## 二、技术架构

- 后端：Spring Boot (Java)，基于 RuoYi 的模块化安全框架。
- 前端：Vue + Element UI，业务页面独立封装，兼容主流浏览器。
- 数据库：基于 MySQL，表结构明晰，便于扩展。
- 安全：Shiro/自定义注解，精准权限控制（如接口 @PreAuthorize 标注）。

---

## 三、核心业务模块

### 1. 学生信息管理
- 对象表名：`student`（学生信息表）
- 字段示例：学号、课程数量、任务完成数、视频/测验/作业/考试完成数、章节学习、讨论、发帖/回帖统计、积分、抢答、评分等。
- API：
    - `GET /system/student/list` 查询学生列表
    - 支持多条件、排名、动态统计检索。
- 典型字段与含义来源：[AiAssistantServiceImpl_draft.java L84-104](https://github.com/Zemtay/TeachingDataManagementSystem/blob/53f6ba0eefbea296a0ca5e1310c64e3bffe92c22/ruoyi-system/src/main/java/com/ruoyi/system/service/impl/AiAssistantServiceImpl_draft.java)

### 2. 课程信息管理
- 对象表名：`course_info`（课程信息表）
- 字段示例：课程 ID、名称、编号、类型、开设学期、创建人、开设班级数、总学生数等。
- API：
    - `GET /system/courseinfo/list` 列表/批量处理
    - `GET /system/courseinfo/studentCount` 查询各课程学生总数
    - TOP N 创建榜、聚合统计 SQL
- Controller/实体参考：[CourseInfoController.java](https://github.com/Zemtay/TeachingDataManagementSystem/blob/53f6ba0eefbea296a0ca5e1310c64e3bffe92c22/ruoyi-admin/src/main/java/com/ruoyi/web/controller/system/CourseInfoController.java)、[CourseInfo.java](https://github.com/Zemtay/TeachingDataManagementSystem/blob/53f6ba0eefbea296a0ca5e1310c64e3bffe92c22/ruoyi-system/src/main/java/com/ruoyi/system/domain/CourseInfo.java)

### 3. 教学班级管理
- 对象表名：`class_info`（班级信息表）
- 字段示例：班级 ID、所属课程、课程编号、班级学生人数、外部班级标识。
- API：`GET /system/classinfo/list`（批量、条件查询）
- 详细表结构见：[AiAssistantServiceImpl_draft.java L62-76](https://github.com/Zemtay/TeachingDataManagementSystem/blob/53f6ba0eefbea296a0ca5e1310c64e3bffe92c22/ruoyi-system/src/main/java/com/ruoyi/system/service/impl/AiAssistantServiceImpl_draft.java)

### 4. 教务活动&成绩统计
- 对象表名：`course_activity`（学生-班级活动表现表）
- 字段示例：主键 ID、学号、课程名、教学班ID、讨论/发帖/回帖/抢答/评分次数、综合成绩、证书状态等。
- 用于追踪学生在各班级中的具体学习行为和成绩（见：[AiAssistantServiceImpl_draft.java](https://github.com/Zemtay/TeachingDataManagementSystem/blob/53f6ba0eefbea296a0ca5e1310c64e3bffe92c22/ruoyi-system/src/main/java/com/ruoyi/system/service/impl/AiAssistantServiceImpl_draft.java)）

### 5. 认证与权限控制
- 基于 RuoYi `@PreAuthorize` 注解，接口级权限细分（教师/学生/管理者不同访问级别）。
- 用户、角色、菜单、部门等基础管理 API，一应俱全，参考[API文档.md](https://github.com/Zemtay/TeachingDataManagementSystem/blob/53f6ba0eefbea296a0ca5e1310c64e3bffe92c22/API文档.md)。

### 6. 教务统计与分析
- 各对象均支持批量、高级查询、聚合统计接口（如课程/班级学生数统计、成绩分布等）。
- 统计接口参考：[CourseInfoController.java L133-158](https://github.com/Zemtay/TeachingDataManagementSystem/blob/53f6ba0eefbea296a0ca5e1310c64e3bffe92c22/ruoyi-admin/src/main/java/com/ruoyi/web/controller/system/CourseInfoController.java)。

### 7. AI 辅助模块
- 后端有独立 `AiAssistantServiceImpl` 草稿实现：集成对数据结构的结构化描述，并设计接口/开放自然语言分析能力。
- 支持直接教学数据分析、辅助决策、字段抽取、AI问答等功能（见[相关代码 L173-182](https://github.com/Zemtay/TeachingDataManagementSystem/blob/53f6ba0eefbea296a0ca5e1310c64e3bffe92c22/ruoyi-system/src/main/java/com/ruoyi/system/service/impl/AiAssistantServiceImpl_draft.java)）。

## 四、主要业务流程

1. 超级管理员拥有系统管理权限。
2. 教师、教务通过认证进入系统。
3. 教师：可查看授课班级学情数据（任务完成 / 互动 / 成绩）、通过 AI 自然语言查询数据，辅助调整教学策略与开展针对性辅导。
4. 教务：负责教学数据全周期管理（采集 / 清洗 / 存储）、全局教学资源监控统计、精细化权限与数据安全管控。
5. 超级管理员创建用户、角色、菜单，配置权限，监控系统安全与运行状态以及系统参数配置维护。

## 六、前后端分离与接口规范

- 所有接口采用 RESTful 风格。
- 支持条件检索、批量操作、分页、动态高级 SQL 查询。
- 前后端通过统一 `/system/...` 路径通信，接口权限通过注解标记。

---

## 七、与 RuoYi 框架集成说明

- 本项目保留 RuoYi 的组织、角色、菜单、字典、岗位等所有基础能力。
- 各业务对象（学生/课程/班级等）均按 RuoYi 规范开发，继承原有管理平台安全体系。
- 自定义模块在 `ruoyi-system` 目录下，新增业务结构与原RuoYi结构无缝兼容。
- 部分接口需分角色授权，菜单权限配置与原生一致。

---

## 八、适用场景与部署建议

- 适用于高校/中小学/培训机构的智能化教学管理场景。
- 推荐部署于内网云服务器，数据库默认使用 MySQL，支持主流 Linux/Windows 环境。
- 如需二次开发，建议遵循各对象领域模型，统一 API 规范。
